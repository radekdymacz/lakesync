# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in LakeSync, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, please email: **security@nativekloud.com**

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

## Response Timeline

- **Acknowledgement:** within 48 hours
- **Initial assessment:** within 1 week
- **Fix and disclosure:** coordinated with reporter, typically within 30 days

## Supported Versions

| Version | Supported |
|---------|-----------|
| 0.x     | Yes (current development) |

## Scope

This policy applies to the LakeSync codebase and its official packages published under the `@lakesync` npm scope.
